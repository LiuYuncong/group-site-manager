# Group Site Manager

Group Site Manager 是一款专为科研课题组量身定制的静态网站本地管理客户端。基于 CustomTkinter 构建了现代化的图形界面，底层严格遵循 Hugo Page Bundles 规范。通过本工具，课题组导师或成员无需直接编写 Markdown 代码或使用 Git 命令行，即可直观地管理团队主页的内容并一键触发远端自动化部署。

## 🎯 核心特性

* **可视化内容管理**：支持新闻动态、团队成员、发表成果、科研项目等模块的所见即所得管理。
* **Hugo Bundles 规范支持**：自动将内容与封面/头像图片打包为独立的文件夹，保持站点资产结构清晰。
* **文献极速录入**：支持单篇或批量导入 BibTeX，内置作者姓名智能映射，自动将英文姓名归一化为预设的课题组成员中文名。
* **静默 Git 同步**：内置健壮的 Git 引擎，一键拉取 (Pull) 与提交推送 (Commit & Push)，自动处理常见的冲突与凭证问题提示。
* **一键本地预览**：后台静默调度 Hugo Server，实时预览修改效果，不弹 CMD 黑框。

## 💻 运行环境

* **操作系统**：Windows 10 / Windows 11 (最佳支持)
* **Python 版本**：推荐 Python 3.10 或 3.11
* **外部依赖**：
  * Git (需添加到系统 `PATH` 并在 Windows Credential Manager 中配置好目标仓库的凭证)
  * Hugo Extended (用于本地预览，需下载 `hugo.exe` 并添加到系统 `PATH`)

## 🚀 快速开始

### 1. 环境准备
确保已安装 Python 3.10+，并建议在项目目录下创建虚拟环境：
```bash
python -m venv venv
# 激活虚拟环境 (Windows)
.\venv\Scripts\activate
```

### 2. 安装依赖

Bash

```
pip install -r requirements.txt
```

### 3. 运行应用

Bash

```
python main.py
```

*(注意：在生产环境打包时，可使用 `pythonw.exe` 隐藏控制台启动)*

## ⚙️ 配置说明

系统配置默认存储于 `C:\Users\<您的用户名>\.group-site-manager\config.json`。 首次运行后，可直接在 GUI 左侧的 **“全局配置”** 菜单中进行设置。

- `repo_path`: 本地 Hugo 站点所在的绝对物理路径（例如：`D:\github\my-group-site`）。
- `remote_git_url`: 绑定的远端 Git 仓库地址（可选，用于辅助提示）。

## 📖 使用说明

1. **选择模块**：在左侧导航栏选择要管理的栏目（如“发表成果”）。
2. **新建/编辑**：点击右上方的 `➕ 新建`，填写表单。如需上传图片，点击“选择图片...”，系统会自动将其复制并重命名至对应的 Page Bundle 文件夹中。
3. **BibTeX 导入**：在“发表成果”界面，可点击 `📥 批量导入 BibTeX`。请直接粘贴包含一篇或多篇文章的 BibTeX 源码，系统将自动解析并尝试匹配课题组成员姓名。
4. **实时预览**：点击右上角 `🌐 本地预览`，浏览器会自动打开 `http://localhost:1313` 查看本地效果。
5. **发布上线**：确认内容无误后，依次点击 `⬇️ 拉取更新`（防止与他人提交冲突）和 `📝 提交并推送`，触发远端 GitHub/GitLab 的自动部署流程。

## 🛠️ 常见问题与故障排查

- **Git 状态显示红色圆点 / 提示“未配置有效的 Git 仓库”**
  - **解决步骤**：前往“全局配置”，确保“本地仓库路径”选择的是包含了 `.git` 隐藏文件夹的网站根目录，配置完成后点击保存并刷新状态。
- **“远程仓库认证失败” 或 Push 失败**
  - **解决步骤**：本应用不直接处理密码输入。请确保您已在 Windows 凭据管理器中缓存了有效的 Git Token，或者在本地仓库通过 Git Bash 成功执行过一次免密推送。
- **删除科研项目或条目时提示“文件可能正被其他程序占用”**
  - **解决步骤**：在 Windows 中，资源管理器的预览窗格或图片查看器可能会锁定文件。请关闭正在查看该项目文件夹的软件或文件夹窗口后重试。
- **“点击预览后提示未找到 hugo 命令”**
  - **解决步骤**：本工具不自带 Hugo。请从 Hugo 官网下载对应版本，并将其所在的文件夹路径加入到 Windows 的“系统环境变量 -> Path”中。配置完成后需重启本应用。

## 🔒 安全与运维注意

- **日志脱敏**：系统在 `core/git_engine.py` 的底层异常处理中，已防范将包含 Token 的 Git 错误明文直接记录落盘。但仍建议避免在未配置 SSH 的 HTTPS 协议中明文硬编码密码。
- **文件锁防护**：采用原子写入策略防止系统断电导致配置文件损坏。
